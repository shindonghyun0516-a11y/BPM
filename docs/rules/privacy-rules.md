# Privacy Rules

이 문서는 오디오 개인정보 보호 규칙을 정의합니다. 이 프로젝트에서 가장 중요한 안전 기준입니다.

## 1. 기본 원칙

초기 MVP에서는 사용자의 오디오를 저장하지 않는다. 서버로 전송하지도 않는다.

BPM 분석은 사용자의 기기 안에서만 수행한다.

## 2. 금지 사항

다음은 초기 MVP에서 금지한다.

- raw audio 파일 저장
- raw audio blob 저장
- PCM 데이터 저장
- waveform 데이터 영구 저장
- 오디오 서버 업로드
- 오디오를 포함한 API 요청
- 사용자 동의 없는 백그라운드 마이크 유지
- 측정 완료 후 마이크 계속 활성화

## 3. 허용 사항

다음은 허용할 수 있다.

- 브라우저 메모리 안에서의 임시 오디오 분석
- BPM 계산을 위한 임시 수치 데이터
- 오디오가 아닌 결과값 표시
- 사용자가 보는 화면 안에서의 측정 상태 표시

## 4. 마이크 권한 안내 규칙

마이크 권한은 사용자가 가장 민감하게 느낄 수 있는 지점이다. 권한 요청 전후의 안내는 쉬운 말로 작성한다.

- 권한 요청 전, 왜 마이크가 필요한지 설명한다.
- 마이크는 BPM 측정을 위해서만 사용한다고 안내한다.
- 초기 MVP에서는 오디오를 저장하지 않는다고 안내한다.
- 초기 MVP에서는 오디오를 서버로 전송하지 않는다고 안내한다.
- 측정이 끝나면 마이크 입력이 중지된다는 점을 안내한다.
- 권한을 거부해도 사용자를 막다른 상태에 두지 않는다.
- 권한 거부 시 1차 대안으로 탭 보정을 제공한다.
- 권한 거부 시 2차 안내로 브라우저 설정에서 마이크 권한을 다시 허용하는 방법을 제공한다.

권한 요청 전 사용자 문구 예시:

```text
BPM을 자동으로 측정하기 위해 마이크 권한이 필요합니다. 오디오는 저장하거나 서버로 전송하지 않으며, 측정이 끝나면 마이크 입력은 중지됩니다.
```

권한 거부 시 사용자 문구 예시:

```text
마이크 권한이 꺼져 있어 자동 측정을 할 수 없습니다. 대신 박자에 맞춰 탭해서 BPM을 측정할 수 있습니다.
```

## 5. 마이크 중지 규칙

다음 상황에서는 마이크 입력을 중지해야 한다.

- 측정 완료
- 측정 취소
- 분석 오류
- 권한 오류
- 화면 이탈 또는 앱 종료 처리 가능 시점

구현에서는 MediaStreamTrack의 stop 동작을 명확히 호출해야 한다.

## 6. 네트워크 규칙

초기 MVP에서 서버로 보낼 수 있는 데이터는 오디오가 아닌 결과 또는 앱 운영 데이터로 제한한다.

서버 전송 금지 대상:

- audio blob
- raw PCM
- waveform
- microphone recording
- 사용자가 녹음한 원본 데이터

## 7. PM 검수 항목

- 화면에서 개인정보 원칙이 명확히 보이는가
- 측정 후 마이크가 꺼지는가
- 네트워크 요청에 오디오가 포함되지 않는가
- 저장소에 오디오 파일이 남지 않는가
- 권한 요청 전 마이크 사용 이유를 설명하는가
- 권한 거부 시 탭 보정 대안을 먼저 제공하는가

## 8. Harness 후보 검사

나중에 자동 검사로 바꿀 수 있는 항목:

- audio blob 업로드 코드 검색
- localStorage에 오디오 저장 시도 검색
- IndexedDB에 오디오 저장 시도 검색
- fetch 요청에 audio payload가 포함되는지 검사
- MediaStreamTrack.stop 호출 여부 검사
