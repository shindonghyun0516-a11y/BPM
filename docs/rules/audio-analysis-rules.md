# Audio Analysis Rules

이 문서는 BPM 자동 측정의 분석 규칙을 정의합니다. 나중에 harness 자동 검사로 바꿀 수 있도록 명확한 조건으로 작성합니다.

## 1. 기본 측정 정책

- 기본 측정 시간은 15초다.
- 기본 BPM 범위는 10~500이다.
- 측정 결과는 BPM 후보와 신뢰도를 함께 제공해야 한다.
- 절반/두 배 BPM 후보를 별도 정책으로 강제하지 않는다.

## 2. 결과 표시 규칙

BPM 결과는 단일 정답처럼 표시하지 않는다.

좋은 예:

```text
추천 BPM: 124
신뢰도: 높음
후보: 124, 62, 248
```

나쁜 예:

```text
BPM: 124
```

단, 절반/두 배 후보를 무조건 강제로 보여주는 정책은 두지 않는다. 후보는 분석 결과와 신뢰도에 따라 표시한다.

## 3. 신뢰도 규칙

모든 자동 측정 결과에는 신뢰도가 있어야 한다.

신뢰도는 다음 요소를 고려할 수 있다.

- 박자 후보가 얼마나 일정한지
- 후보 BPM이 얼마나 뚜렷한지
- 입력 레벨이 너무 낮지 않은지
- 소음이 너무 크지 않은지
- 15초 동안 충분한 분석 데이터가 있는지

## 4. 불안정 결과 규칙

다음 상황은 불안정 결과로 처리할 수 있다.

- 후보 BPM 간 점수 차이가 작음
- 입력 레벨이 너무 낮음
- 피크 간격이 불규칙함
- 분석 가능한 리듬이 부족함
- 측정 시간이 충분히 채워지지 않음

불안정 결과에는 반드시 다음 행동을 제공한다.

- 다시 측정
- 탭 보정

## 4-1. 낮은 신뢰도 참고 후보 규칙

정규 결과 기준 중 interval stability만 통과하지 못했고, 입력 신호와 박자 후보가 충분한 경우에는 낮은 신뢰도 참고 후보를 표시할 수 있다.

낮은 신뢰도 참고 후보는 다음 원칙을 지켜야 한다.

- 기존 안정 결과가 "박자 간격이 너무 불규칙함" 때문에 실패한 뒤에만 fallback으로 검토한다.
- 입력 신호, onset 후보, BPM 후보가 부족하면 표시하지 않는다.
- 입력 신호 충분성은 RMS 기반 energy, signal variation, signal detected count를 함께 보고 판단한다.
- BPM 후보가 기존 V0 BPM 범위 밖이면 표시하지 않는다.
- 후보 분산이 지나치게 크면 표시하지 않는다.
- UI 라벨은 "추천 BPM"이 아니라 "참고 BPM 후보"로 표시한다.
- 신뢰도는 반드시 "낮음"으로 표시한다.
- 문구에는 정확한 BPM으로 확정하지 말고 참고 후보로만 확인하라는 의미가 있어야 한다.
- 2&4 또는 1&3 강세, 재즈/블루스/스윙/피아노 솔로처럼 interval이 흔들리는 입력을 위한 보조 정책이다.
- 정규 비트 곡의 안정 결과를 약화하는 방식으로 사용하지 않는다.

## 5. 범위 제한 규칙

10 미만 또는 500 초과 BPM은 MVP 기본 결과로 확정하지 않는다.

범위를 벗어난 계산값은 다음 중 하나로 처리한다.

- 후보에서 제외
- 낮은 신뢰도 처리
- 불안정 결과 처리

## 6. PM 검수 항목

- 결과가 숫자 하나만 확정적으로 보이지 않는가
- 신뢰도가 같이 표시되는가
- 불안정 결과에 다음 행동이 있는가
- 15초 측정 정책이 지켜지는가
- BPM 범위 10~500 정책이 지켜지는가

## 7. Harness 후보 검사

나중에 자동 검사로 바꿀 수 있는 항목:

- 측정 기본값이 15초인지 검사
- BPM 최소값이 10인지 검사
- BPM 최대값이 500인지 검사
- 결과 타입에 confidence가 있는지 검사
- unstable 상태에서 retry/tap correction CTA가 있는지 검사
