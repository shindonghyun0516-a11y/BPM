# MVP SCOPE

이 문서는 첫 버전에 포함할 것과 제외할 것을 명확히 나누는 문서입니다.

## 1. MVP 목표

첫 버전의 목표는 "모바일웹에서 마이크로 BPM을 자동 측정하고, 후보 BPM과 신뢰도를 보여주는 것"이다.

완벽한 음악 분석 앱을 만드는 것이 아니라, 자동 측정의 기본 경험과 개인정보 보호 원칙을 검증한다.

## 2. MVP에 포함할 기능

### 모바일웹 첫 화면

- 앱 목적을 짧게 설명
- 측정 시작 버튼 제공
- 오디오 저장/전송을 하지 않는다는 안내 제공

### 마이크 권한 요청

- 사용자가 측정을 시작할 때 마이크 권한 요청
- 권한 거부 시 대체 안내 제공

### 15초 자동 측정

- 기본 측정 시간은 15초
- 측정 중 진행 상태 표시
- 측정 취소 가능

### BPM 후보 결과

- BPM 후보를 표시
- 각 후보 또는 추천값에 신뢰도 표시
- 기본 BPM 범위는 10~500

### 불안정 결과 안내

- 결과가 낮은 신뢰도일 때 불안정 상태로 표시
- 다시 측정 CTA 제공
- 탭 보정 CTA 제공

### 탭 보정

- 사용자가 버튼을 탭해 BPM을 간단히 보정
- 자동 측정 결과와 탭 보정 결과를 구분해 표시
- MVP에서는 고급 편집 기능 없이 "탭으로 다시 계산한 BPM"만 제공

### 마이크 중지

- 측정 완료 시 마이크 중지
- 측정 취소 시 마이크 중지
- 오류 발생 시 마이크 중지

### 기본 검증

- 모바일 화면 확인
- 권한 거부 플로우 확인
- 오디오 저장/전송 금지 확인
- 측정 후 마이크 중지 확인

## 3. MVP에서 제외할 기능

- 네이티브 모바일앱 출시
- 사용자 로그인
- 클라우드 저장
- 측정 기록 저장
- 오디오 파일 업로드
- 녹음 파일 저장
- 서버 기반 오디오 분석
- 음악 제목 인식
- 장르 분석
- 박자표 분석
- 소셜 공유
- 유료 결제
- 관리자 페이지

## 4. 나중에 검토할 기능

- 모바일앱 확장
- 측정 기록 저장
- 사용자가 동의한 경우의 로컬 기록
- 더 자세한 BPM 후보 설명
- 자동 측정 결과와 탭 보정 결과의 상세 비교
- 다양한 음악 장르별 튜닝
- 더 긴 측정 모드
- 접근성 향상 모드

## 5. 범위 변경 규칙

MVP 범위를 변경하려면 다음을 먼저 확인한다.

- 개인정보 정책에 영향이 있는가
- 오디오 저장/전송 정책을 바꾸는가
- 개발 기간이 크게 늘어나는가
- 자동 측정 핵심 경험보다 부가 기능이 우선되는가
- PM이 명시적으로 승인했는가

PM 승인 없이 MVP 범위를 넓히지 않는다.

## 6. MVP에서 탭 보정 범위

탭 보정은 자동 측정이 불안정할 때 사용자가 직접 박자를 눌러 대략적인 BPM을 확인하는 보조 기능이다.

MVP에 포함:

- 탭 버튼
- 최근 탭 간격 기반 BPM 계산
- 탭 결과의 신뢰도 또는 안정성 표시
- 자동 측정 결과와 탭 결과를 구분해서 보여주기

MVP에서 제외:

- 탭 기록 저장
- 탭 패턴 상세 분석
- 자동 측정 결과와 탭 결과의 복잡한 비교 리포트
- 사용자가 BPM 후보를 수동 편집하는 고급 기능
