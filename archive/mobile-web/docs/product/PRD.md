# PRD: BPM Automatic Measurement

## 1. 제품 목적

BPM 자동 측정 앱은 사용자가 음악을 틀어놓으면 모바일 기기의 마이크로 BPM을 자동 추정하는 도구다.

기존 Tap Tempo 방식은 사용자가 버튼이나 스페이스바를 박자에 맞춰 계속 눌러야 한다. 이 프로젝트는 그런 수동 입력 없이 음악을 듣고 BPM 후보를 자동으로 계산하는 것을 목표로 한다.

## 2. 해결하려는 문제

사용자는 음악의 BPM을 알고 싶지만, 기존 방식에는 불편함이 있다.

- 박자에 맞춰 정확히 탭해야 한다.
- 사용자가 탭을 흔들리게 하면 결과도 흔들린다.
- 모바일에서 스페이스바 기반 도구를 쓰기 어렵다.
- BPM 결과가 하나의 정답처럼 보이지만 실제로는 불확실할 수 있다.
- 주변 소음이나 리듬이 약한 음악에서는 측정 결과를 신뢰하기 어렵다.

## 3. 대상 사용자

- 음악을 듣고 BPM을 빠르게 알고 싶은 일반 사용자
- DJ, 댄서, 운동 콘텐츠 제작자
- 음악 연습 또는 편집을 하는 사용자
- 모바일에서 간단히 BPM을 확인하고 싶은 사용자

## 4. 핵심 가치

- 자동 측정: 사용자가 박자에 맞춰 계속 누르지 않아도 된다.
- 신뢰도 표시: 결과가 얼마나 믿을 만한지 함께 보여준다.
- 후보 표시: BPM을 단일 정답처럼 보여주지 않는다.
- 개인정보 보호: 초기 MVP에서는 오디오를 저장하거나 서버로 보내지 않는다.
- 모바일 우선: 설치 없이 모바일웹에서 먼저 사용할 수 있다.

## 5. MVP 주요 기능

### 측정 시작

사용자가 측정 시작 버튼을 누르면 마이크 권한을 요청한다.

### 자동 BPM 측정

기본 15초 동안 마이크 입력을 분석한다.

### BPM 후보 표시

결과는 하나의 숫자만 보여주지 않고, 후보 BPM과 신뢰도를 함께 보여준다.

### 신뢰도 표시

각 후보 또는 최종 추천 BPM에는 신뢰도가 있어야 한다.

### 불안정 결과 안내

측정이 불안정하면 다시 측정 또는 탭 보정을 안내한다.

### 탭 보정

자동 측정이 어렵거나 사용자가 결과를 보정하고 싶을 때 탭으로 BPM을 보정할 수 있다.

### 마이크 중지

측정 완료, 취소, 오류 발생 시 마이크 입력을 중지한다.

### 권한 거부 안내

마이크 권한이 거부되면 사용자가 무엇을 해야 하는지 안내한다.

## 6. MVP 제외 범위

- 오디오 저장
- 오디오 서버 전송
- 사용자 계정
- 클라우드 기록 저장
- 음악 파일 업로드
- 장르 분석
- 자동 곡 인식
- 소셜 공유
- 유료 결제
- 네이티브 모바일앱 출시

## 7. 고정 정책

- 기본 측정 시간은 15초다.
- 기본 BPM 범위는 10~500이다.
- 절반/두 배 BPM 후보를 별도 정책으로 강제하지 않는다.
- BPM 결과에는 신뢰도를 표시한다.
- 불안정 결과는 다시 측정 또는 탭 보정을 안내한다.
- 초기 MVP에서는 오디오 저장을 금지한다.
- 초기 MVP에서는 오디오 서버 전송을 금지한다.
- 측정 후 마이크 입력을 중지한다.
- 권한 거부 시 대체 안내를 제공한다.

## 8. 성공 기준

- 사용자가 모바일웹에서 측정을 시작할 수 있다.
- 측정 시작 후 15초 기준으로 결과가 나온다.
- 결과에 BPM 후보와 신뢰도가 표시된다.
- 불안정한 측정 결과가 별도 상태로 표시된다.
- 사용자가 다시 측정 또는 탭 보정을 선택할 수 있다.
- 마이크 권한 거부 시 안내가 표시된다.
- 측정 후 마이크가 중지된다.
- 오디오가 저장되거나 서버로 전송되지 않는다.

## 9. PM 검수 기준

- 사용자가 앱의 목적을 즉시 이해하는가
- 측정 중 상태가 명확한가
- 결과가 과도하게 확정적으로 보이지 않는가
- 낮은 신뢰도 결과가 솔직하게 안내되는가
- 개인정보 보호 원칙이 화면과 동작에 반영되는가
